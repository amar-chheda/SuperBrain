Act 02
The Paradigm Shift
Slides 05–09 · 5 slides
05 · AGENTIC
Agentic AI isn't a chatbot — it's orchestrated decision-making.
The model is not the product. The workflow is the product. Closer to distributed systems than data science.
reframe
06 · RESPONSIBILITY
Foundation models shifted the responsibility — they didn't eliminate it.
Old: accuracy at model level. New: accuracy at system level. Prompt design, retrieval, validation.
key insight
07 · TRAINING
How small language models get made.
Distillation → pruning → quantization → LoRA. The funnel from 70B to 3.8B.
technical
08 · GENERALIST/SPECIALIST
Generalist or specialist? Both. Route.
LLMs for open-ended reasoning. SLMs for classification, extraction, routing. Don't pick — orchestrate.
model selection
09 · LATENCY
Latency is a system design constraint — not UX.
phi3:mini classification: 0.5–1.5s. llama3.1:8b generation: 4–12s. Latency compounds in agentic loops.
latency table
Act 03
Why Small Language Models?
Slides 10–16 · 7 slides
10 · WHY SLMs
Why I chose small, local models.
Privacy · Cost · Forcing function · Observability. The four reasons.
overview
11 · COST
What running LLMs actually costs.
GPT-4o: $7,500+/mo at 1K articles/day. llama3.1:8b local: ~$12 (electricity). Enterprise LLM: $500K–2M/yr.
numbers
12 · NVIDIA PAPER
NVIDIA's case: SLMs are the future of agentic AI.
Berkeley Function Calling Leaderboard. SLMs match LLMs on agentic tasks. arXiv:2506.02153.
research
13 · OPEN-WEIGHTS
The open-weights moment.
Llama 2 → Mistral → Phi-3 mini → Llama 3.1 → Phi-4. The timeline that made Superbrain possible.
timeline
14 · BALANCE
SLMs and LLMs — orchestrated.
Orchestra metaphor. Gartner: 3× task-specific model adoption vs LLMs by 2027.
architecture
15 · FORCING FUNCTION
Small models force better system design.
Precise prompts · Explicit boundaries · Validated output · Engineered fallbacks. Before/after examples.
core insight
16 · DIVISION OF LABOUR
Division of labour.
90% cost reduction on repetitive tasks. "How do I route and compose multiple model sizes?" — InfoWorld 2024.
enterprise
Act 04
What I Built — Superbrain
Slides 17–22 · 6 slides
17 · SYSTEM
Superbrain at a glance.
Entry: API/Bot/CLI. Core: ingestion/classify/QA/digest. Storage: Postgres + pgvector. Three local models. Zero cloud AI.
architecture
18 · MODELS
Three local models. Four agentic roles.
llama3.1:8b (QA + chunk + digest) · phi3:mini (classify, 4× faster) · nomic-embed-text (768-dim).
model table
19 · CHUNKING AGENT
The chunking strategy agent.
LLM chooses: semantic / recursive / fixed. JSON output. Deterministic fallback to "fixed" on any failure.
agent pattern
20 · CLASSIFICATION
Classification by integer index — hallucination prevention by design.
Topics as integers [1][2][3], not UUIDs. Unknown IDs silently dropped. Bad confidence → coerced to "low".
hallucination guard
21 · QA PIPELINE
Grounded QA with evidence gating.
Evidence gate: ≥2 chunks AND RRF score ≥ 0.005. SOURCES line. Chunk ID validation vs evidence set.
pipeline
22 · RRF
Reciprocal rank fusion.
score = Σ 1/(60+rank). Vector + BM25, one Postgres. No separate vector DB.
algorithm
Act 04.5
What Actually Broke — The Honest Part
Slides 22-BRIDGE–22D · 5 slides · ALL NEW
22-BRIDGE · INTERLUDE
Let me tell you about the things that broke.
Bold typographic transition. Previews all 4 failure stories as cards. Sets emotional register: "failures are more educational than successes."
transition · tone-setter
22A · LFM STORY
The model we planned to use wasn't ready.
LFM2-7B was the planned digest model. Wasn't stable in Ollama. Still a comment in the codebase: "swap to lfm2-7b when available."
model availability
22B · STRUCTURED OUTPUT
When phi3:mini talked back.
"very_high" confidence. Markdown fences wrapping JSON. {"matches": [...]} object wrapper. Three defensive fixes required.
structured output
22C · ASYNC LIFECYCLE
The session that wouldn't stay open.
APScheduler + SQLAlchemy async. Three failed attempts. Final: closure in main.py with fresh session per execution.
async bug
22D · GPT-4O TRAP
GPT-4o taught us the wrong architecture.
GPT-4o spike "just worked." Switched to llama3.1:8b → hallucinated IDs, thin-evidence answers. Forced: evidence gate + SOURCES + ID validation.
architecture gap
Act 05
Lessons Learned
Slides 23–28 · 6 slides
23 · LESSON 01
Agentic AI is orchestration, not autonomy.
Dream: one LLM, endless arrows. Reality: directed graph of bounded steps with explicit error paths.
lesson 1
24 · LESSON 02
The prompt is the contract.
Every prompt: role + task + output schema + what to do if you can't. Before/after comparison.
lesson 2
25 · LESSON 03
Constraints are features in disguise.
"If your system only works with a powerful model, you haven't built a system — you've built a prompt."
lesson 3 · key quote
26 · LESSON 04
Observability is not optional.
Every LLM call logged. p50/p95/p99 metrics. Eval harness: recall@k, citation presence, groundedness.
lesson 4 · metrics
27 · LESSON 05
Retrieval quality is answer quality.
80% of QA improvement budget → retrieval. Hybrid + RRF + per-chunk embeddings. Generator is the easy part.
lesson 5 · 80%
28 · LESSON 06
Local-only changes the design space.
Privacy is a feature. Air-gapped environments. No rate limits. No per-token bill. Compliance without footnotes.
lesson 6
Act 06
Takeaways + Close
Slides 29–32 · 4 slides
29 · MENTAL MODEL
A mental model for local-first agentic AI.
5-layer stack: Entry → Orchestration → LLM calls → Retrieval → Storage. One responsibility per layer.
model
30 · CHECKLIST
LLM or SLM — a checklist.
Repetitive/structured/fast → SLM. Reasoning/creative → LLM. Combining both → always orchestrate.
decision framework
31 · THREE THINGS
Three things I'd tell past me.
Design prompts before code. Pick smallest model that works. Build eval harness FIRST, not last.
memorable
32 · CLOSE
Agentic AI is still engineering. It just has different seams.
Final quote. Resources: Superbrain repo, arXiv:2506.02153, InfoWorld, ollama.ai. Q&A.
closer
