The chunking strategy agent — LLM-as-decision-maker.

app/application/ingestion/chunking_agent.py
# 1. Ask the model. Give it a closed action space.
prompt = """
Decide the chunking strategy. Output JSON only:
{"strategy": "semantic"|"recursive"|"fixed"}
Article (first 2000 chars): ..."""

# 2. Trust the response — only after parsing.
try:
    strategy = json.loads(response)["strategy"]
    assert strategy in {"semantic", "recursive", "fixed"}
except (JSONDecodeError, KeyError, AssertionError):
    strategy = "fixed"  # ← deterministic fallback

Classification by integer index — hallucination prevention by design.

In the prompt
[1] Machine Learning
[2] Security
[3] DevOps
[4] Databases
↓ phi3:mini ↓
Model output
[{"topic_id": 1, "confidence": "high"},
{"topic_id": 99, "confidence": "high"}]
↓ parser ↓
After index_map resolution
1 → a1f4-c2… ML ✓ kept
99 → no match ✗ dropped silently
Topics are passed to the LLM as integers, not UUID strings.
▸ Unknown IDs → dropped silently
▸ Bad confidence ("certain") → coerced to "low"
▸ Markdown fences stripped before JSON parse
Lesson Structure the prompt so hallucination is physically impossible, not just statistically unlikely.

Reciprocal rank fusion — when vector and keyword disagree.

VECTOR · COSINE
#1 · chunk_A
#2 · chunk_B
#3 · chunk_C
#4 · chunk_D
+
BM25 · TSVECTOR
#1 · chunk_E
#2 · chunk_F
#3 · chunk_A
#4 · chunk_G
=
FUSED · RRF
chunk_A · .0323
chunk_B · .0164
chunk_E · .0164
chunk_F · .0161
Formula
score(c) = Σ 1 / (60 + ranki)
Worked example · chunk_A
vector #1 → 1/(60+1) = .0164
BM25 #3 → 1/(60+3) = .0159
total: .0323
Why this matters A chunk that shows up in both rankings gets a real boost. Postgres does vector + FTS — no separate vector DB.

Superbrain · A. ChhedaFailure 01 / What broke
Act 4 · What actually broke

When phi3:mini talked back.

What we got back
FAILURE 1 · confidence outside the schema
[{"topic_id": 1, "confidence": "very_high"}]
# → ValueError: invalid enum value
FAILURE 2 · json wrapped in markdown fences
```json
[{"topic_id": 1, "confidence": "high"}]
```
# → json.JSONDecodeError
FAILURE 3 · object wrapper instead of bare array
{"matches": [{"topic_id": 1...}]}
# → TypeError: list indices must be int
The defensive fixes shipped
FIX 1 · confidence coercion
Any value ∉ {high, medium, low} → coerced to "low"
FIX 2 · fence stripping
Strip ```json ... ``` before JSON parse
FIX 3 · wrapper detection
Check for {"matches": [...]} → extract array
The reframe The model wasn't broken. Our prompt wasn't tight enough. Every fix was fixing the contract, not the model.
Superbrain · A. ChhedaFailure 02 / What broke
Act 4 · What actually broke

The session that wouldn't stay open.

ATTEMPT 1 · build use case at startup
# startup: session created once, stored
use_case = GenerateDailyDigestUseCase(session)
7am: sqlalchemy.exc.InvalidRequestError
session already closed
ATTEMPT 2 · factory at job time
async with get_session() as session:
    use_case = DigestUseCase(session)
# context manager exits → session closes
await use_case.execute(...)  # ← boom
DetachedInstanceError inside execute()
ATTEMPT 3 · closure in main.py ✓
async def _run_digest(target_date, triggered_by):
    async with get_session_factory()() as s:
        # session lives for the entire call
        uc = GenerateDailyDigestUseCase(s, ...)
        return await uc.execute(target_date, ...)
APScheduler fires at 7am UTC.
Job touches the database.
Session is already dead.
▸ Three attempts, each failed differently
▸ Root cause: async resource lifetime isn't intuitive
▸ Fresh session per execution is always safer
▸ The pattern already existed in our API background tasks
Lesson Small models don't cause all your bugs. Async does.

LESSON 02 · The prompt is the contract.

BEFORE · vague
"Classify this article."

# What the model does:
# - Returns prose paragraphs
# - Wraps in markdown sometimes
# - Invents categories
# - Random capitalisation
# You parse free text. You lose.
AFTER · contract
# role + task + schema + fallback
"""You are a content classifier.
Topics (use integer id):
[1] ML  [2] Security  [3] DevOps

Output JSON:
[{"topic_id": <int>,
  "confidence": "high"|"medium"|"low",
  "reason": "<one sentence>"}]

Return [] if no match."""
The rule Every prompt specifies: the role, the task, the output schema, and what to do if you can't complete it.

Act 5 · Lessons learned

LESSON 05 · Retrieval quality is answer quality.

80%
of your QA improvement budget should go to retrieval.
the generator is the easy part
What paid off
+ Hybrid retrieval (vector + BM25)
+ RRF fusion across the two
+ Per-chunk embeddings, not per-article
+ Postgres FTS index on chunk content
What didn't matter much
· Prompt length tweaks
· Temperature tuning
· Switching to a larger model for generation
Garbage in, hallucination out. The LLM can only cite what it was given.
